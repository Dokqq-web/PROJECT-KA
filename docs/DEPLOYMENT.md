# Развёртывание QA Bot

## Требования

- Docker Engine 26+ и Docker Compose.
- 2 CPU и 4 GB RAM для одного браузера.
- Дополнительные ресурсы на каждый параллельный Chromium.
- Отдельный Appium-агент для мобильных тестов.
- macOS, Xcode и Appium XCUITest для реальных iOS-запусков.

## Подготовка

1. Скопируйте `.env.example` в `.env`.
2. Создайте длинный случайный `BOOTSTRAP_API_KEY`.
3. Создайте 32-байтный `SECRET_MASTER_KEY` в base64.
4. Не добавляйте `.env` в систему контроля версий.

## Запуск

```text
docker compose up --build -d
```

Панель будет доступна на порту `8080`. Нажмите «Ключ доступа» и введите
`BOOTSTRAP_API_KEY`. SQLite и артефакты находятся в именованных Docker-томах.

## TLS и внешний доступ

Не публикуйте порт `8080` напрямую в интернет. Разместите перед сервисом
reverse proxy с HTTPS, ограничением размера запроса и сетевым allowlist или VPN.
Для рабочих сред рекомендуется отдельный домен, например `qa.internal.example`.

## Appium

Appium запускается на отдельном агенте рядом с устройствами. В панели
зарегистрируйте endpoint агента и capabilities. Для Android обычно используется
`appium:automationName=UiAutomator2`, для iOS — `XCUITest`.

## Резервное копирование

Останавливать сервис для SQLite WAL-совместимой резервной копии безопаснее всего:

1. Остановить контейнер.
2. Скопировать том `qa-bot-data`.
3. Скопировать том `qa-bot-artifacts`, если нужны скриншоты.
4. Запустить контейнер.

Мастер-ключ секретов должен резервироваться отдельно. Без него расшифровать
сохранённые секреты невозможно.

## Обновление

1. Сделайте резервную копию.
2. Получите новую версию исходного кода.
3. Выполните `docker compose up --build -d`.
4. Проверьте `/health`, очередь и один smoke-тест.
